## 2️⃣ 파생 변수 생성 (add_features 함수)

```python
def add_features(frame: pd.DataFrame) -> pd.DataFrame:
    out = frame.copy()
    out['channel_4_flag']    = (out['registered_via'] == 4).astype(int)      # ①
    out['high_skip_flag']    = (out['avg_skip_rate'] > 0.30).astype(int)     # ②
    out['low_explore_flag']  = (out['trend_unq_7d_vs_30d'] < 0.80).astype(int)  # ③
    out['has_logs']          = out['avg_skip_rate'].notna().astype(int)      # ④
    out['auto_high_risk']    = out['channel_4_flag']                         # ⑤
    out['gender_M']          = (out['gender'] == 'male').astype(int)         # ⑥
    return out
```

### 📊 생성 변수 상세 정보

| # | 변수명 | 정의 | 타입 | 용도 |
|---|--------|------|------|------|
| ① | `channel_4_flag` | `registered_via == 4` | 이진 | **✅ FEATURE_COLS 포함** (모델 학습) |
| ② | `high_skip_flag` | `avg_skip_rate > 0.30` | 이진 | ❌ 메타데이터만 (분석용, 모델X) |
| ③ | `low_explore_flag` | `trend_unq_7d_vs_30d < 0.80` | 이진 | ❌ 메타데이터만 (분석용, 모델X) |
| ④ | `has_logs` | `avg_skip_rate.notna()` | 이진 | **✅ FEATURE_COLS 포함** (모델 학습) |
| ⑤ | `auto_high_risk` | `= channel_4_flag` | 이진 | ❌ 메타데이터만 (분석용, 모델X) |
| ⑥ | `gender_M` | `gender == 'male'` | 이진 | **✅ FEATURE_COLS 포함** (모델 학습) |

## 3️⃣ 모델 입력 변수 (FEATURE_COLS - 11개)

```python
FEATURE_COLS = [
    'avg_skip_rate',              # 행동: 평균 스킵률
    'trend_unq_7d_vs_30d',        # 행동: 탐색 트렌드
    'avg_complete_rate',          # 행동: 완성률
    'std_num_unq_last_30d',       # 행동: 곡 다양성
    'channel_4_flag',             # ✅ 생성됨 (파생①)
    'tenure_at_tx',               # 결제까지 기간
    'is_discounted',              # 할인 여부
    'cum_cancel_count',           # 취소 횟수
    'is_method_changed',          # 수단 변경 여부
    'has_logs',                   # ✅ 생성됨 (파생④) - 결측값 flag
    'gender_M',                   # ✅ 생성됨 (파생⑥)
]
```

### ⚠️ 중요: 모델이 학습하는 변수 vs 학습하지 않는 변수

**모델 O (RandomForest에 입력):**
- `avg_skip_rate`, `trend_unq_7d_vs_30d`, `avg_complete_rate`, `std_num_unq_last_30d`
- `channel_4_flag` ← **파생①, 모델에서 자동 가중치 학습**
- `tenure_at_tx`, `is_discounted`, `cum_cancel_count`, `is_method_changed`
- `has_logs` ← **파생④, 결측값 존재 여부**
- `gender_M` ← **파생⑥**

**모델 X (학습하지 않음, 메타데이터로만 사용):**
- `high_skip_flag` ← **파생②** (분석용 규칙, 모델은 `avg_skip_rate` 원본 사용)
- `low_explore_flag` ← **파생③** (분석용 규칙, 모델은 `trend_unq_7d_vs_30d` 원본 사용)
- `auto_high_risk` ← **파생⑤** (현재 미사용)

## 5️⃣ 단계별 데이터 흐름

```
Step 1: 데이터 로드
    ↓
    TARGET_COLUMNS (14개) 로드

Step 2: 피처 엔지니어링 (add_features)
    ↓
    6개 파생 변수 생성
    ├─ 모델 입력: ①④⑥
    └─ 메타데이터: ②③⑤

Step 3: 데이터셋 준비 (prepare_dataset)
    ↓
    FEATURE_COLS (11개)만 X로 추출
    메타데이터는 meta로 별도 반환

Step 4: 모델 학습 (RandomForest)
    ↓
    X (11개 피처) → y (is_churn_cycle)

Step 5: 스코어링 (score_user)
    ↓
    ML 점수 60% + 룰 부스트 40%
    ├─ ML 점수: 모델이 channel_4_flag 포함하여 학습
    └─ 룰 부스트:
       - registered_via == 4 → +40
       - avg_skip_rate > 0.30 → +20
       - trend_unq_7d_vs_30d < 0.80 → +20
```

## 🔴 논리적 문제점

### 문제 1: 채널4의 이중 처리

```python
# 문제 상황
FEATURE_COLS에 'channel_4_flag' 포함
├─ 모델 학습 시: RandomForest가 자동으로 channel_4_flag의 가중치 학습
└─ 피처 중요도: channel_4_flag가 상위에 나타남

+ 스코어링 시: registered_via == 4 → +40점 추가 부스트
└─ Result: "이중 패널티" 발생!
```

**보고서의 표현:** "모델 없이도 적용" (독립적 룰로 표현)
**실제 코드:** 피처로 포함 + 별도 부스트 (이중 처리)

### 문제 2: 비사용 파생 변수

```python
'high_skip_flag'  # 생성되지만 FEATURE_COLS에 미포함
'low_explore_flag'  # 생성되지만 FEATURE_COLS에 미포함
'auto_high_risk'  # 생성되지만 코드에서 사용 안 함
```

→ 메타데이터로만 생성되고 해석용으로만 사용

## 📋 최종 결론

### 현재 코드의 설계

1. **모델 학습 단계**: FEATURE_COLS (11개) 사용
   - `channel_4_flag` ✅ 포함 → 모델이 자동 가중치 학습

2. **점수 계산 단계**: ML + 규칙 결합
   - ML 스코어: 60% 가중치
   - 규칙 부스트: 40% 가중치
     - `registered_via == 4` 기반 +40점
     - `avg_skip_rate > 0.30` 기반 +20점
     - `trend_unq_7d_vs_30d < 0.80` 기반 +20점

### 문제점 요약

| 문제 | 현재 상황 | 권고 방향 |
|------|---------|---------|
| 채널4 이중 처리 | 피처 + 부스트 | 둘 중 하나만 선택 (피처는 제거, 룰 기반으로 단순화) |
| 비사용 변수 | `high_skip_flag`, `low_explore_flag` 생성되지만 미사용 | 메타데이터로 명확히 표기 또는 제거 |
| 보고서 불일치 | "모델 없이도 적용" vs 실제는 피처 포함 | "모델 학습 + 추가 규칙 부스트" 명시 |
