## <a name="stage-0"></a>**Stage 0: 데이터 전처리 및 피처 엔지니어링**

### 📁 산출물
- **파일**: DuckDB 파이프라인 → `data/df_track1_final_master.parquet`
- **규모**: 약 2,154만 행 × 다수 피처
- **단위**: 유저별 **결제 회차(tx)** 1행 구조

### 🔍 핵심 코드 해석

#### **1. 결제-로그 슬라이딩 윈도우 방식**

```sql
-- 각 결제(tx)마다 "그 결제 만료 30일 직전" 로그 수집
-- 예: 결제 만료 2017-03-31이면 → 2017-03-01~2017-03-31 로그 집계
SELECT 
  msno,
  membership_expire_date,
  -- 만료 30일 전부터 만료일까지 로그 수집
  AVG(log.avg_skip_rate) as avg_skip_rate,
  AVG(log.num_unq) as num_unq_30d
FROM transactions tx
LEFT JOIN user_logs log 
  ON tx.msno = log.msno 
  AND log.date BETWEEN DATE_SUB(tx.membership_expire_date, 30) 
                   AND tx.membership_expire_date
```

**해석**:
- ✅ **장점**: 이탈 예측 시점과 동일하게 "만료 30일 전 행동"을 캡처
- ✅ **타이밍 일치**: 실제 갱신 의사결정이 이루어지는 시점의 데이터 사용
- ⚠️ **주의**: 결제 회차별 단위이므로, 유저별 스냅샷으로 축소 필요

#### **2. 핵심 파생 변수 설명**

| 변수명 | 계산식 | 의미 | 활용 |
|--------|--------|------|------|
| `avg_skip_rate` | (num_25 + num_50) / total_songs | 25% 이상 재생 못한 비중 | **이탈 신호**: 높을수록 불만족 |
| `trend_unq_7d_vs_30d` | unq_7d / unq_30d | 최근 탐색 다양성 추세 | **권태기 신호**: <0.8이면 흥미 감소 |
| `avg_complete_rate` | num_100 / total_songs | 완청률 | **만족도 proxy**: 높을수록 좋음 |
| `is_churn_cycle` | 만료 후 30일 내 재구독 없음 | **타겟 변수** | 0/1 이진 분류 |
| `channel_4_flag` | registered_via == 4 | 채널 4 여부 | **자동 고위험군** |

**특이사항**:
- `avg_skip_rate`와 `trend_unq_7d_vs_30d`에 **결측률 높음** (~15~20%)
  - 이유: 로그 없는 비활동 유저 또는 청취 기간 부족
  - 처리: ML 모델에서 0으로 임퓨테이션 (비활동=고위험)

## <a name="stage-1-3"></a>**Stage 1~3: EDA 및 코호트/생존 분석**

### 📁 산출물
- **파일**: `tobigs 코호트퍼널패스생존.ipynb`
- **실행 결과**: 다수의 시계열 차트, 생존곡선, 패스 분석 지표

### 🔍 핵심 분석

#### **1. 코호트 히트맵 분석**

```
행: 가입 월(cohort_month)
열: 구독 수개월(subscription_month)
값: 해당 코호트의 n번째 개월 이탈률
```

**발견사항**:
- **2016년 10월 코호트**: M+1 이탈률 **45%** (극도로 높음)
- **2017년 이후**: M+1 이탈률 안정적 (25~30%)
- **해석**: 2016년 10월 가입자들이 특히 취약했던 이유 불명확 → **비즈니스 리뷰 필요**

#### **2. 퍼널 분석 (3단계 전환)**

```
┌─ Stage 1: 첫 결제 이후 "첫 30일" ─→ 40.1% 탈락 (초기 이탈)
│
├─ Stage 2: 3개월 안정 (결제 3회 이상) ─→ 36.5% 탈락 (권태기 이탈)
│
└─ Stage 3: 활성 구독자 ─→ 23.4% 탈락 (이탈)
```

**핵심 지표**:
- **초기 이탈/권태기 비율**: 약 55:45
- **통계 검증**: 이항검정 p<0.05 (유의)
- **전략적 함의**: 온보딩 개선 효과가 가장 큼

#### **3. 패스 분석 (Path Analysis)**

```
온보딩 경로별 스킵률:
  - 일반 유저: avg_skip_rate = 0.35
  - 이탈 유저: avg_skip_rate = 0.52 (+17.5% 높음) ★

권태기 경로별 탐색 트렌드:
  - 활성 유저: trend_unq_7d_vs_30d = 0.95
  - 이탈 유저: trend_unq_7d_vs_30d = 0.91 (-4% 낮음)
```

**해석**:
- 스킵률이 온보딩 단계에서 **가장 강력한 이탈 신호**
- 탐색 다양성 감소는 권태기 신호이지만 영향도는 약함

#### **4. Kaplan-Meier 생존분석**

```
비교군:
  - 코호트 A (2016-01): 12개월 생존율 62%
  - 코호트 B (2016-10): 12개월 생존율 48%
  
Log-rank test p-value: 0.0012 (유의)
→ 코호트별 생존 패턴에 **통계적으로 유의한 차이** 존재
```

**해석**: 가입 시점이 단순 우연이 아니라 **구조적 차이** 암시

#### **5. 채널 분석 (Channel 4 고위험군)**

```
채널 4 (registered_via = 4):
  - 이탈율: 89.9% (매우 높음!)
  - 일반 채널: 이탈율 26.8%
  - 차이: **63.1% 포인트** (압도적)

Log-rank test: p<0.05 (유의)
→ 채널 4는 자동 고위험군으로 분류 가능
```

**전략적 함의**: 채널 4 유입 최소화 또는 특별 온보딩 필요

#### **1. 피처 엔지니어링**

```python
def add_features(frame: pd.DataFrame) -> pd.DataFrame:
    out = frame.copy()
    out['channel_4_flag'] = (out['registered_via'] == 4).astype(int)
    out['high_skip_flag'] = (out['avg_skip_rate'] > 0.30).astype(int)
    out['low_explore_flag'] = (out['trend_unq_7d_vs_30d'] < 0.80).astype(int)
    out['has_logs'] = out['avg_skip_rate'].notna().astype(int)
    out['auto_high_risk'] = out['channel_4_flag']  # 자동 플래깅
    out['gender_M'] = (out['gender'] == 'male').astype(int)
    return out
```

**변수 해석**:
- `channel_4_flag`: 채널 4 여부 → **가장 강력한 신호**
- `high_skip_flag`: 스킵률 >30% → **온보딩 단계 신호**
- `low_explore_flag`: 탐색 트렌드 <0.8 → **권태기 신호**
- `has_logs`: 활동 로그 존재 여부 → **비활동 구분**
